Core Terminology

TermFull FormCategoryDefinition
LPNLicense Plate NumberEntityUnique barcode label on every physical carton or tote. The fundamental tracking unit for all warehouse inventory.
iLPNInbound License Plate NumberEntityVendor carton arriving from supplier. Lifecycle: Expected → Received → Putaway → Reserve.
oLPNOutbound License Plate NumberEntityShipping carton going to customer. Lifecycle: Allocated → Picked → Packed → Weighed → Manifested → Loaded → Shipped.

SlugSorter Processing GroupConceptSequenced, capacity-controlled group of totes processed together by the sorter. Core throughput control mechanism.

SpurConveyor Diversion LaneEntityBranch lane off the main conveyor where cartons are physically diverted. Types: parcel, live-load, LTL, reject, QC.
VASValue Added ServicesProcessSpecial processing before shipment: gift wrapping, compliance labeling, kitting, inspection. Reject Flag = Y while open.

STX / ETXStart/End of TextProtocolControl chars wrapping every TCP socket message. STX = 0x02 (start), ETX = 0x03 (end). Required on all connections.
ACK / NAKAcknowledgement / Negative AckProtocolACK confirms successful receipt. NAK signals rejection with reason. NAK triggers retry and alert.
HBHeartbeatProtocolPeriodic keepalive from WES to WCS/Sorter every 5 seconds. Missing HB_ACK marks device OFFLINE immediately.

Message Architecture

WMS → WES → WCS (Real-Time)

100
ASN File
WMS → WES (sFTP)
Pre-advise inbound shipment. Contains iLPN list, items, weights, dimensions.
101
LPN Destination (DEST)
WMS → WES → WCS
Routes an iLPN to reserve/putaway or oLPN to a target lane. Primary routing instruction.
102
Verify ASN (ASNVERIFY)
WMS → WES
Confirms all iLPNs in an ASN have been received. Closes ASN verification loop.
103
Wave Download
WMS → WES (sFTP)
Releases wave. Primary driver of slug creation. Contains all tasks, LPNs, VAS codes, S* task types.
105
Tote Pick Complete (ZPK)
WMS → WES
Signals tote closure. WES maps tote to slug and fires 6A2 to Sorter.
106
Pack Complete (PACKED)
WMS → WES (pipe-delimited)
oLPN fully packed. Contains weight, labels, reject flag, mode, pallet group, tracking #. Only pipe-delimited message.
107
VAS Complete
WMS → WES
VAS processing completed. WES clears reject flag and releases carton to conveyor.
114
Adjust / Split
WMS → WES
oLPN contents changed — unit added, removed, split, or cancelled. WES updates routing.
116
Consume Tote
WMS → WES
Tote empty and ready to re-enter picking loop. Triggers 6A4 Tote Complete to Sorter.
117
Pallet Group Update (PGUPDATE)
WMS → WES (event 9030)
Updates pallet group assignment via manual load planning.
118
Pallet Group Assign
WMS → WES (event 9031)
Assigns spur to pallet group or handles routing request for PG.
119
Pallet Group Unassign
WMS → WES (event 9032)
Removes spur assignment on yard move or shipment close.

WCS → WES → WMS (Real-Time)

200
Container Status (SCANNED)
WCS → WES → WMS
iLPN barcode read at receiving scanner. Includes actual weight, dimensions, reason code.
201
Route Request (ROUTINGREQ)
WCS → WES → WMS
WCS cannot determine routing. Requests WMS decision. WMS responds with 101 DEST.
202
IB Pallet Build (IBPLT)
WCS → WES → WMS
Pallet built at receiving sorter after grouping iLPNs. WMS assigns pallet location.
208
Manifest Confirm
WCS → WES → WMS
oLPN enters manifest lane. WES assigns tracking number. WMS status = Manifested.
209
Loaded Confirm
WCS → WES → WMS
oLPN loaded to truck. WMS marks shipment closed. Status = Shipped.
211
Locate LPN / Divert Confirm
WCS → WES → WMS
LPN diverted. Sent except for recirculation/reject. Confirms physical destination.

Sorter Messages (Fixed-Width)

6A2
Picked LPN Directive
WES → Sorter
Associates tote with slug on 105 receipt. Contains Slug ID, tote count, carton count, priority, routing.
6A3
Shipping Carton Complete
WES → Sorter
Forwarded from 106 Packed. Notifies Sorter oLPN is packed and ready for final sortation.
6A4
Tote Complete
WES → Sorter
Forwarded from 116 Consume Tote. Tells Sorter tote is empty. Triggers slug closure tracking.
6R2
Recovery Directive
WES → Sorter
Replay of 6A2 directives during recovery. Restores Sorter routing queue after directive loss.
6
Divert Confirmation
Sorter → WES
Sorter confirms tote physically diverted to correct lane. Closes 6A2 directive.
7
Recovery Initiation
Sorter → WES
Sorter signals directive loss. WES replays 6A2s. Auto-closes after 10s of no directives.

Weight Validation Logic

AT
Absolute Tolerance
Weight outside fixed absolute bounds regardless of expected. → QC Spur
RT
Relative Tolerance
Weight deviates from expected by more than configured %. → QC Spur
DS
Diverted Successfully
Both checks passed. Normal divert to target lane confirmed.
UN
Unknown Carton
Barcode not found in WES/WMS. → Exception Spur
PG
Unknown Pallet Group
Pallet group missing. WES requests routing decision from WMS.
LF
Lane Full
Target lane at capacity. WES requests alternate lane from WMS.
MD
Missing Dimensions
Dimensioner failed. Route to exception for manual measurement.
05

Slug Building

A slug is a sequenced, capacity-controlled group of totes processed together by the sorter. Slugs prevent buffer overflow, maintain processing order, and balance lane utilization. Core throughput control mechanism for the entire outbound flow.

📦
Standard Slug
2 → MAX CONFIGURED TOTES
Multiple totes within size limit. Grouped by wave / carrier / zone / cutoff. Default processing mode.
1️⃣
One-of-One Slug
EXACTLY 1 TOTE
Single tote. Full-case or direct-ship orders. Triggered via 101 LPN Destination. No grouping wait.
🔗
Multi-Tote Slug
2+ TOTES, SAME ORDER
Multiple totes from a single customer order. All must complete before routing proceeds.
📮
Batch Slug
SHARED ROUTING
Totes sharing same carrier, destination, or cutoff. Grouped for sorter efficiency and reduced traversal.
⚠️
Oversize Slug
EXCEEDS CONFIGURED MAX
Flagged for alternate routing. Alert raised. Changing max size requires empty sorter buffers — operational window required.

Slug Algorithm

1

Parse Wave Download File

WES reads the 103 Wave file. Identifies ITM records with Task Type. Task Types starting with S* require slug grouping. Others are processed as one-of-one.

2

Group Tasks

Group tasks by carrier, zone, cutoff, and destination. One task = one picking LPN/tote. Multi-tote orders grouped together.

3

Enforce Max Slug Size

Groups within limit → Standard Slug. Groups exceeding limit → flagged as Oversize, alert raised, alternate routing applied.

4

Assign Slug ID & Metadata

Unique Slug ID assigned. Track: Wave ID, tote count, carton count, priority, process codes, batch ID, oversize flag.

5

Wait for Tote Pick Complete (105)

WES waits. When 105 ZPK received, maps tote LPN to assigned Slug ID. Updates slug tote completion count.

6

Send 6A2 to Sorter

Fixed-width 6A2 Picked LPN Directive sent to Sorter. Contains Slug ID, # totes, # cartons, priority, routing, process codes, last location.

7

Track Slug Closure

All oLPNs packed (106) + all totes consumed (116) → WES sends 6A4 Tote Complete → Slug marked CLOSED. Wave closure checked when all slugs closed.

Error Handling & Recovery

Error ConditionDetectionAction
Invalid message formatWES ValidationNAK returned. Alert raised. Retry logic engaged.
Sequence number mismatchWES ValidationNAK returned. Alert raised. Message quarantined.
Duplicate message (same Control ID)WES ValidationMessage discarded. ACK returned (idempotent accept).
Missing required fieldWES ValidationNAK with field name returned. Alert raised.
Heartbeat timeout (WCS or Sorter)WES TimerDevice marked OFFLINE. Equipment down alert to WMS. No messages forwarded to offline device.
Unknown LPN at inboundWCS ScannerReason code UN. WCS routes to exception spur. WMS creates manual receive task.
Weight tolerance failureWCS ScaleReason code AT or RT. WCS routes to QC spur. WMS flags for inspection.
Lane full at divertWCSReason code LF. WES requests alternate lane from WMS via 201. WMS responds with 101.

Sorter Recovery Protocol

1

Sorter Detects Directive Loss

After restart or communication failure. Sorter sends Recovery Initiation (msg 7) to WES.

2

WES Looks Up Replay Queue

WES queries all active slugs with outstanding directives not yet confirmed by Sorter (no msg 6 received).

3

WES Sends Recovery Directive (6R2)

All outstanding 6A2 Picked LPN Directives replayed via 6R2 Recovery Directive to Sorter.

4

Sorter Resumes Processing

Processes replayed directives. Routes totes to correct buffer lanes and pack lanes.

5

Auto-Close After 10 Seconds

Sorter recovery session auto-closes after 10 seconds of no additional directives. Normal operation resumes.

⬥ Recovery does NOT affect WMS inventory. WMS status remains unchanged throughout recovery. Only the Sorter's routing directive queue is restored.
